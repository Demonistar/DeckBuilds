        Summary: Top-level dict with key="Finance", display_name, description,
        version, deck_api_version="1.0", home_category="Professional",
        secondary_categories, entry_function="register", tab_definitions.

        Summary: workspace.tabs contains entries for slots 1, 2, 3 with id,

[ ] A07. Contract returns three module panel tabs

SECTION D. APPEND-ONLY LEDGER BEHAVIOR
================================================================================

[ ] D01. New transaction writes ledger row
        Summary: Quick Add saves a new row. recorded_at is set to now.
        amount is positive, direction tells sign.

[ ] D02. No ledger row is ever deleted
        Summary: Delete action sets voided=true with voided_at and
        void_reason. Original row remains in table.

[ ] D03. Edit writes supersede row, not UPDATE
        Summary: Editing a row in Ledger view creates a new row with
        supersedes_id pointing at original. Original is unchanged.

[ ] D04. ledger_edits row written on every edit
        Summary: Each edited field gets one ledger_edits row with field
        name, old value, new value, reason.

[ ] D05. Queries use latest non-superseded, non-voided row
        Summary: All aggregations and views filter out superseded rows and
        voided rows by default.

[ ] D06. "Show edit history" filter reveals chains
        Summary: Toggle in Ledger view shows original + all supersedes
        chronologically with edit reasons.

[ ] D07. "Show voided" filter reveals voided rows
        Summary: Voided rows displayed struck-through and dimmed when
        filter on.

[ ] D08. Undo void available for 7 days
        Summary: Voided rows show "Undo void" button for 7 days after
        voiding. After that, button gone but row still in DB.

[ ] D09. Settings changes write to settings_history
        Summary: Every budget threshold change, every retirement input
        change, every method change writes a new settings_history row.

[ ] D10. "Current setting" query is time-aware
        Summary: Reading current value uses
        WHERE effective_from <= now ORDER BY effective_from DESC LIMIT 1.

[ ] D11. Account balance computed not stored
        Summary: Account balance = opening_balance + sum of ledger rows
        for that account minus pending checks. Never read from a stored
        column.
